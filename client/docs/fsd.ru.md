# Правила FSD (Feature-Sliced Design)

## Основные принципы

### 1. Иерархия слоев
- **Нельзя импортировать из нижележащих слоев в вышележащие**
- Порядок слоев (сверху вниз): `app` → `pages` → `widgets` → `features` → `entities` → `shared`
- Верхние слои могут импортировать только из нижних слоев

### 2. Изоляция слайсов
- **Слайсы не могут импортировать друг друга**
- Слайсы на одном уровне должны оставаться независимыми
- Взаимодействие между слайсами должно происходить через нижние слои

### 3. Публичный API
- **У каждого слайса есть точка входа** (`index.ts`)
- Все импорты из слайса должны идти через его публичный API
- Детали внутренней реализации скрыты

### 4. Стандартизация сегментов
- Стандартные сегменты: `ui`, `model`, `api`, `lib`, `config`, `consts`
- Единообразная структура во всех слайсах

## Дополнительные правила

### 5. Слабая связанность, сильная связность
- Модули должны быть слабо связаны между собой
- Группируйте связанную функциональность вместе

### 6. Изоляция бизнес-логики
- Бизнес-логика должна находиться в `entities`, `features` или `processes`
- UI-компоненты должны быть максимально простыми

### 7. Философия слоя Shared
- `shared` содержит переиспользуемый код без бизнес-логики
- По возможности должен быть независим от проекта
